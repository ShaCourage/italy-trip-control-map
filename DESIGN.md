# Italy Trip Control Map — 설계 문서

> 2026-06-19 ~ 06-28 로마·피렌체 여행 (여성 2인, 6/24 로마→피렌체 이동)
> iPhone 홈 화면에 추가해 쓰는 PWA. "여행 당일 5초 안에 다음 행동을 알 수 있게."
> 마지막 갱신: 2026-06-14

---

## 1. 제품 원칙

모든 기능·데이터·화면 결정은 아래 3개 원칙으로 판단한다.

### P1. 객관성 — 지어낸 수치 금지
- 평점·리뷰수·가격대 등 **외부 수치는 사람이 확인해 넣은 값만** 표시한다. 확인 안 된 곳은
  내부 `추천 점수(rank)`로 표시하고, "Google 평점은 실시간 확인 권장"을 명시한다.
  (※ 초기 버전은 공식으로 평점을 생성해 "Google 4.5"로 표시했음 — 제거 완료, 재도입 금지)
- 모든 장소는 `sourceIds`(출처)를 가진다. 확인값에는 확인일(`lastChecked`)을 붙인다.
- 추정치는 추정임을 라벨로 드러낸다. 예: "혼잡 높음(예상)", "도보 감각"(직선거리 기반).
- 주관 지표(여자 2인 적합도, 사진 점수, 관광객 함정 가능성)는 유지하되 **내부 편집 점수임을
  UI에서 구분**한다. Google·공식 데이터처럼 보이게 하지 않는다.

### P2. 커스텀 — 데이터와 코드 분리, 사용자 상태는 전부 저장
- 콘텐츠(장소·일정·회화·팁)는 `src/data/`의 타입 있는 데이터 파일로만 존재한다.
  화면 코드에 장소명·점수 하드코딩 금지.
- 사용자가 화면에서 바꿀 수 있는 것(루트, 완료 체크, 체크리스트, 설정)은 모두
  localStorage에 버전 키로 영속화한다. JSON 백업/복원을 제공한다.
- 코드 수정 없이 바꿀 수 있어야 하는 것: 숙소 좌표, 시작 탭, 기본 모드, 내 장소(커스텀 핀).

### P3. 현장 우선 — 5초 원칙
- 첫 화면에서 "지금 어디로, 얼마나 걸리고, 뭘 조심하는지"가 바로 보인다.
- 모든 장소에서 2탭 안에 Google Maps 길찾기가 열린다.
- 오프라인(지하철·로밍 끊김)에서도 일정·팁·회화는 보인다.

## 2. 메뉴 구조 (확정)

하단 5탭. 이름과 책임을 다음과 같이 정리한다.

| 탭 | 역할 | 비고 |
|---|---|---|
| **지도** | 메인. 핀 + 오늘 루트 + 루트 편집 + 상황 추천 | 첫 화면 |
| **오늘** | 당일 대시보드: 다음 목적지, 거리/도보, 당일 체크 | 지도와 같은 day 상태 공유 |
| **일정** | 6/19–28 전체 흐름, 날짜별 내 코스 vs 추천안, 플랜 B | |
| **장소** | (현 "랭킹"에서 개명) 장소 DB 탐색: 도시/카테고리/검색/권역 | "랭킹"은 순위만 보여줄 것 같은 인상 — 탐색+추가가 본 기능 |
| **더보기** | 아래 5섹션으로 통합 | |

**더보기 섹션 통합** (현행 7개 → 5개):

| 신규 섹션 | 흡수하는 현행 섹션 | 이유 |
|---|---|---|
| 안전·꿀팁 | 안전 + 한국인 | 둘 다 "조심할 것/알아둘 것"이라 사용자 입장에선 한 주제 |
| 음식 가이드 | 음식 | 유지 |
| 회화 | 회화 | 유지 (긴급 회화는 안전·꿀팁에서도 링크) |
| 체크리스트 | 체크 | 유지 |
| 데이터·설정 | 지도파일 + 출처 + **설정(신규)** + **백업(신규)** | 관리성 기능을 한곳에 |

분리하지 않는 것: "오늘"과 "지도"를 합치자는 안이 있었으나, 지도는 *조작*(루트 편집),
오늘은 *읽기*(브리핑) 화면으로 목적이 달라 유지한다.

## 3. 화면 구성

### 3.1 지도 (메인)
```
[날짜 스트립]  [모드 ▼]  [필터 칩들]
┌─────────────────────────┐
│  지도 (핀 + 루트 선 + 번호)  │
│  [선택 장소 플로팅 요약 바]   │
└─────────────────────────┘
[다음 목적지 + 길찾기 버튼]
[오늘 권역 / Must / 먹고쉬기 / 주의 신호 카드]
[선택 장소 상세 카드]
[추천 코스 카드 (적용/빠진 곳 추가)]
[오늘 루트 리스트 (완료/순서/삭제, 잠금 표시)]
[지금 추천 (모드 반영)]
```
- 핀 = 카테고리 색 + 짧은 한글 라벨. 루트에 포함되면 순서 번호 배지.
- 잠긴 일정(예약·기차·숙소)은 자물쇠 표시, 이동·삭제·교체 불가.
- 장소 추가는 항상 "숙소 복귀(밤)" 앞에 삽입.

### 3.2 오늘
다음 목적지 히어로 → 거리/도보/주의/Must 메트릭 → 오늘 루트 타임라인(완료 체크)
→ 상황별 후보 3개 → 당일 체크리스트(영속).

### 3.3 일정
날짜 카드 리스트. 카드마다: 내 코스 요약, 추천안 요약, 플랜 B(fallback), Maps 링크,
"추천안 적용"/"직접 편집(→지도)".

### 3.4 장소 (현 랭킹)
도시 세그먼트 + 카테고리 칩 + 검색 + 권역 묶음 → 장소 카드 리스트.
카드: 확인된 Google 평점 *또는* 추천 점수(구분 표시), 소요시간, 사진/안전/예약,
태그, [지도에서 보기] [루트 추가] [Maps].

### 3.5 더보기
섹션 탭 5개(§2). **데이터·설정**에 신규 기능:
- 설정: 로마/피렌체 숙소 좌표(지도 탭에서 현재 선택 핀으로 지정 가능), 시작 탭, 기본 모드
- 백업: 내 상태(루트·체크·설정) JSON 내보내기/가져오기
- 지도파일: CSV/KML (Google My Maps용)
- 출처: 전체 출처 리스트 + 확인일

## 4. 디렉터리 구조 / 공통 모듈 (리팩터링 목표)

현재 `App.tsx` 단일 파일 2,100줄 + `data.ts`/`extraData.ts` 이원화. 아래로 분리한다.

```
src/
  App.tsx                 # 탭 셸 + 전역 상태만 (<300줄 목표)
  screens/
    MapScreen.tsx
    TodayScreen.tsx
    PlanScreen.tsx
    PlacesScreen.tsx      # 현 RankingScreen
    MoreScreen.tsx
  components/             # 2개 이상 화면에서 쓰는 것만 승격
    DayStrip.tsx
    PlaceCard.tsx         # 장소 카드 (장소 탭/추천/검색 공용)
    PlaceInsight.tsx      # 상세 카드
    PlaceThumb.tsx
    Pill.tsx / IconButton.tsx
    MapView.tsx           # Leaflet 래핑 (마커/루트선/맞춤)
  lib/
    geo.ts                # haversine, 거리 포맷, 도보 환산
    score.ts              # getPlaceScore, 추천 점수, 모드별 가중치
    urls.ts               # Google Maps 장소/길찾기 URL
    export.ts             # CSV/KML/JSON 백업
    storage.ts            # 버전 키 load/save, 마이그레이션
    routes.ts             # RouteItem 조작(추가/교체/이동/잠금 규칙) 순수 함수
  hooks/
    useRoutes.ts          # routes + done + 조작 함수 (현재 App.tsx 내 로직)
    usePersistentRecord.ts# localStorage Record<string,boolean>
    usePlaceMedia.ts      # 위키 이미지 (모듈 캐시 포함)
  data/
    catalog.ts            # 장소/출처 데이터 단일 진입점
    schema.ts             # 타입 전부 (Place, TripDay, …)
    labels.ts             # cityLabels, categoryLabels, 색상
    places/rome.ts        # 장소: 도시별 분리 (data.ts + extraData.ts 통합)
    places/florence.ts
    enhancements.ts       # 확인된 평점/이미지/하이라이트 (lastChecked 필수)
    days.ts               # tripDays
    phrases.ts / safety.ts / guides.ts / sources.ts / checklist.ts
```

규칙:
- **routes.ts의 루트 조작은 순수 함수**로 작성해 화면과 분리 (잠금/숙소 앞 삽입 규칙이
  3곳에 중복돼 있는 현재 문제 해소).
- `score.ts` 외부에서 평점·점수 문자열을 조립하지 않는다 (객관성 라벨 일원화).
- 데이터 파일은 빌드 시 `validatePlaces()`로 검증: id 중복, pairWith 깨진 참조,
  sourceIds 누락, 좌표 범위.

## 5. 데이터 스키마 (변경점)

기존 `Place` 유지 + 객관성/커스텀 필드 추가:

```ts
type Place = {
  // ...기존 필드 유지...
  sourceIds: string[];        // 필수 (비면 빌드 검증 실패)
};

type PlaceEnhancement = {
  google?: {
    rating: number;
    reviewCountLabel: string;
    lastChecked: string;      // [추가] "2026-06-10" — UI에 "6/10 확인" 표시
    // ...
  };
};

// [신규] 사용자 상태 (localStorage, JSON 백업 대상)
type UserState = {
  version: number;
  routes: Record<string, RouteItem[]>;
  done: Record<string, boolean>;
  checks: Record<string, boolean>;
  settings: {
    romeHotel?: { lat: number; lng: number; label: string };
    florenceHotel?: { lat: number; lng: number; label: string };
    startTab: TabKey;
    defaultMode: ModeKey;
  };
  customPlaces: Place[];      // 사용자가 추가한 핀
  notes: Record<string, string>; // placeId → 메모
};
```

localStorage 키는 `italy-trip-state-v3` 하나로 통합하고 `storage.ts`가 v2 키들
(`custom-routes-v2`, `done-v1`, `checks-v1`)을 1회 마이그레이션한다.
